Release Notes

4.0.0

Release Date: 28 January 2025

  • Bug Fix: Compound nodes are now matched by name instead of array index position. Previously, when importing translated content, nodes could be matched to wrong targets if ordering differed between source and target documents. Translation imports now correctly identify compound nodes regardless of order.
  • Performance Improvement: Refactored DefaultJcrContentNodeBinder to use name-based indexing, reducing complexity from O(C×J) to O(J+C). This provides up to 10x faster binding for documents with many child nodes.
  • Breaking Changes - Removed Methods:
    • DefaultJcrContentNodeBinder#getCompoundNodeNames()
    • DefaultJcrContentNodeBinder#getContentNodeBindingTargetSelector()
    • DefaultJcrContentNodeBinder#setContentNodeBindingTargetSelector()
    • DefaultJcrContentNodeBinder#findChildNodesByNameAndType()
    These methods were using index-based matching which caused the translation import bug. The new implementation uses name-based indexing via NodeIndex helper class.

1.3.0

Release Date: 24 October 2018

  • HIPFORGE-210: Add DefaultJcrContentNodeBinder#setSubNodesMergingOnly supporting the merge of existing subnodes so their properties are kept if these not present in the incoming data.
  • HIPFORGE-213: Bump Jackson2 version from 2.4.5 to 2.8.11.

1.2.0

Release Date: 2 May 2018

  • HIPFORGE-154: DefaultJcrContentNodeBinder#bind will now delete all compound subnodes when the binding concerns a document. This replaces deletion of subnodes based on the incoming POJO, which fails if none are left in a multiple or optional compound structure.
  • Better extendability for the DefaultJcrContentNodeBinder.
